# ClearPrice — Houston Oncology Drug Pricing Intelligence

A local, single-user tool for looking up sourced, fully-cited pricing
breakdowns for 33 oncology drugs across Houston-metro hospitals — hospital
charges, CMS ASP/WAC benchmarks, Medicare reimbursement, 340B acquisition
estimates (where verified), per-payer negotiated rates, and a payer-driven
cross-hospital comparison view. Every number shown carries a citation back to
its source (hospital MRF, CMS file, FDA label, HRSA 340B registry); a missing
number is always shown as missing, never estimated or averaged.

See `.specify/memory/constitution.md` for the non-negotiable rules this
project is built against (no assumptions, double-checked verification, full
provenance on every rendered number).

## Stack

- **Backend**: Python 3.11, FastAPI + uvicorn — ingestion pipeline,
  calculation engine, and a read-only JSON API
- **Frontend**: Plain HTML/CSS/JS, no build step
- **Data**: Normalized per-hospital JSON files under `houston_hospitals/`
  (git-ignored — generated by ingestion, not checked in)

## Project layout

```text

├── src/

│   ├── reference_data/  # 33-drug FDA dose/ASP/WAC records, 43-hospital list
│   ├── verification/    # 340B enrollment + payer-scheme double-checks
│   ├── calc/            # pricing/margin formulas, breakdown assembler
│   └── api/             # FastAPI routes
└── tests/                # contract / integration / unit tests

frontend/                # Hospital page + Payer Comparison page
docker/                  # Dockerfile + docker-compose for the backend
specs/001-houston-oncology-pricing/  # Spec Kit artifacts (spec/plan/tasks/contracts/data-model)
```

## Running it

```bash
# 1. Bring up the backend
docker compose -f docker/docker-compose.yml up --build

# 2. Run ingestion (manual, no scheduler — re-run any time to refresh)
docker compose -f docker/docker-compose.yml exec backend python -m src.ingestion.run_all

# 3. Confirm every hospital ingested
curl http://localhost:8000/api/status
```

`/api/status` reports ingestion completeness (`total_hospitals`,
`ingested_success/failed`, `not_yet_ingested`, `complete`) — since
`houston_hospitals/` isn't checked into git, this is how you confirm
ingestion actually ran for every hospital before using the app.

Then open the frontend (served by the backend at `http://localhost:8000/`).

Full walkthrough, including provenance-enforcement tests and manual
per-hospital re-ingestion: `specs/001-houston-oncology-pricing/quickstart.md`.

## Scope

Fixed to 33 named oncology drugs and the Houston-metro hospitals locked in
`specs/001-houston-oncology-pricing/spec.md`. Not user-editable in this
feature; expansion to other drugs/metros is out of scope for v1.
