 ClearPrice Oncology Intelligence — Build Methodology How the 33-drug MRF intelligence dataset was built, and how to read it A walk through the actual pipeline — hospital MRF → HCPCS classification → CMS ASP overlay → WAC/340B research overlay — followed by one fully worked example (Pembrolizumab / Keytruda) showing every number and where it came from. source: 74110155_the-methodist-hospital_standardcharges.json + CMS July 2026 ASP Payment Limit File + 340B_pricing_research.md 1 · The pipeline Six stages, run in order, each stage's output verified against the previous stage before moving on. Stages 1–4 build the sourced dataset; stages 5–6 turn it into the dose-scaled, scheme-correct profit/loss read shown in full for J9271 in §4 and §5. Nothing downstream of stage 1 introduces a number that didn't already exist in a cited source or a formula stated in the intelligence report. 01 Ingest the hospital MRF Loaded the hospital's published CMS-schema machine-readable file and counted every code_information entry typed HCPCS . The file carried 20,463 HCPCS-typed line items across 14,651 distinct codes. A source-data defect was found and logged rather than silently fixed: HCPCS code c1751 appeared lowercase in the original file — normalized to C1751 and noted in the output file's notes field. in: standardcharges.json (68.7 MB) out: hcpcs_only_standardcharges.json 02 Classify into oncology categories For each of 7 oncology categories, identified the 5 most-market-used drugs via research (not by hospital charge amount), then checked whether that exact HCPCS code existed in the hospital's own file. Where a top-market drug's code was absent, it was excluded, not substituted — the earlier draft that swapped in an alternate code (e.g. J9070 → J8530) was rejected and rebuilt. Two category slots stayed empty rather than filled with a stand-in. in: hcpcs_only_standardcharges.json out: oncology_top5_by_category.json (33 items) 03 Overlay CMS Medicare Part B ASP Downloaded the live CMS July 2026 Medicare Part B Payment Limit File directly from cms.gov and matched all 33 HCPCS codes against it. 32 of 33 matched the main payment-limit file; 1 (J9202, Zoladex) matched the separate "Drugs Not Payable Under Part B" file instead — confirmed, not assumed, that it is not separately reimbursed that quarter. in: cms.gov/files/zip/july-2026-medicare-part-b-payment-limit-files.zip out: oncology_top5_asp_reference.json 04 Overlay WAC / list-price research NADAC was dropped on request (retail-pharmacy-only benchmark, not applicable to physician-administered infusion drugs). True 340B ceiling price is confirmed unobtainable — HRSA's OPAIS database is login-gated by law. In its place: a WAC/list/cash-price benchmark per drug, sourced and cited per row. One figure (Epogen, J0885) was cross-checked by downloading the manufacturer's actual pricing PDF directly, after a search-summary snippet looked too synthesized to trust at face value. in: manufacturer price-disclosure PDFs, Drugs.com Price Guide, HRSA / eCFR statutory reference out: 340B_pricing_research.md — 33 of 33 rows sourced 05 Scale to a full dose and compute margin Every rate above is natively per-mg (J9271's fixed HCPCS billing unit). To be usable, it has to be scaled to a real dose (200 mg, FDA-labeled standard adult Keytruda dose — a public prescribing-label fact, not a value present anywhere in the hospital's MRF) and netted against a cost basis: Medicare reimbursement (ASP + 6%) minus estimated 340B acquisition cost (ASP − 27%, the COA report's average industry discount, not this drug's actual confidential 340B ceiling price). Every scale-up and every margin line is labeled calculated, distinct from the sourced figures it's built from — demonstrated in full for J9271 in §4. in: per-mg hospital/ASP/WAC rates, 200mg dose scale, ASP+6% / ASP−27% formulas (report §2) out: per-dose profit/loss verdict, per payer line 06 Map the result onto its reimbursement scheme The margin math above only applies as-is to Part B buy-and-bill drugs like J9271. The product has to route each drug to the scheme that actually governs its payment before the margin means anything: Part B buy-and-bill (ASP+6%, or ASP+3% in reduced settings), Part A MS-DRG bundled payment for CAR-T/CGT ($269,139–$314,231 base vs. $373,000–$475,000 acquisition cost — the category the report's documented losses come from, not this one), commercial (2–7× Medicare, per-contract), or Medicaid (state-inconsistent, CGT Access Model for cellular/gene therapy). Full scheme detail and figures in §5. in: oncology_category, reimbursement scheme per drug out: scheme-correct margin/loss statement, not a one-formula-fits-all number 2 · What came out of it Every figure below is a count from the built files, not a projection. 20,463 HCPCS line items in source MRF 14,651 distinct HCPCS codes 33 drugs matched across all 3 sources 7 / 7 oncology categories represented 3 · The 7 categories, as built Each category was scoped to 5 market-leading drugs by research; the count shown is how many of those 5 the hospital's own MRF actually carried.
